## 🛠️ **第二阶段：技术架构优化与工程化升级（2–3周）**

### 当前技术栈：
- **前端**：React + Tailwind CSS + Chart.js + Axios  
- **部署**：Vercel（SSR/静态）  
- **状态管理**：React Context（建议升级）  
- **API**：模拟数据（无真实后端）  

### ✅ 优化目标：**可扩展、可测试、可维护、高性能**

| 模块 | 改造方案 | 原因 |
|------|----------|------|
| **状态管理** | 从 `Context` → **Zustand**（轻量、高性能、TypeScript友好） | Context 在复杂页面中性能差，Zustand 无渲染开销 |
| **API 层** | 抽离为 `src/api/`，封装 `useFetch` Hook，支持 mock/real 切换 | 便于后期对接真实行情接口（如 Tushare、JoinQuant） |
| **路由** | 从手动页面切换 → **React Router v6** | 更规范，支持 deep link、页面缓存 |
| **响应式优化** | Tailwind CSS 优化为 `mobile-first`，使用 `sm:`, `md:`, `lg:` 媒体查询统一规范 | 避免重复样式，提升可维护性 |
| **性能优化** | 图表懒加载、图片 CDN 压缩、字体预加载、代码分割 | Vercel 首屏加载时间目标：**<1.5s**（Lighthouse ≥ 90） |
| **代码规范** | 引入 ESLint + Prettier + Husky + Commitlint | 保证团队协作代码质量 |
| **测试** | 增加 Cypress E2E 测试（登录、交易、退出） | 防止上线后功能崩溃 |

> ✅ **工程化成果**：  
> - `src/` 目录结构清晰：  
> ```
> src/
> ├── components/       # 可复用组件（按钮、卡片、图表）
> ├── pages/            # 页面（Login, Home, Trade, Profile）
> ├── api/              # 接口封装（mock + real）
> ├── hooks/            # 自定义 Hook（useAuth, useMarketData）
> ├── store/            # Zustand 状态管理
> ├── assets/           # 图片、图标、字体
> ├── utils/            # 工具函数（格式化金额、时间）
> └── styles/           # 全局样式、主题变量
> ```
